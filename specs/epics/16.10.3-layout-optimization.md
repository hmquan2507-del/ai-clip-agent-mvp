## Resizable system

- **`useLayoutResizer`** (`layout-resizer.tsx`): on `pointerdown`, captures the starting pointer coordinate and current value, then attaches `pointermove`/`pointerup` listeners on `window` for the duration of the drag (self-cleaning on `pointerup`). Computes `next = clamp(startValue + (invert ? -delta : delta), min, max)` and calls `onChange`. No `ResizeObserver` was needed — dragging computes size directly from pointer deltas, which is the standard, lower-overhead technique for this exact interaction (drag-to-resize, as opposed to observing externally-driven size changes) and satisfies the "or equivalent" clause in the brief.
- **`PanelDivider`** (`panel-divider.tsx`): the visual, focusable (`tabIndex=0`, `role="separator"`) bar consuming `useLayoutResizer`. Supports horizontal (width) and vertical (height) orientation, `invert` (for dividers whose panel sits to the drag-origin side, e.g. Inspector/Timeline), double-click to reset to the default value, and arrow-key resizing for keyboard accessibility (Shift = larger step).
- **`useDesktopEditorLayout`** (`hooks/use-desktop-editor-layout.ts`): the single source of truth for all four dimensions plus four collapse flags, all wrapped in one `useMemo` so `DesktopGrid` (itself `React.memo`-wrapped) only re-renders when a dimension or collapse flag actually changes — not on every runtime tick (playhead position, pending-command flags, etc., which change far more often and flow through separately as panel *content* props, not layout props).
- **Collapsing**: each collapsible panel renders its own compact/slim view when `collapsed` — a 40px vertical strip with a rotated label for Asset Library/Inspector, a 64px horizontal strip for Timeline, and an icon-only 48px rail for Tool Rail's compact mode — rather than disappearing, so a `PanelCollapseButton` is always reachable to restore it.

## Known limitations

- Panel size/collapse state is component-local (`useState`, not persisted) — reloading the page resets every panel to its default size. Persistence (e.g. `localStorage`) wasn't requested this sprint.
- Timeline's "prepare for AI Track / Video / Subtitle / Overlay / Music / Voice / SFX" requirement is satisfied at the layout level only (the panel has real height/collapse behavior and a header strip ready for track-type chips) — no track rows were added, per the explicit "Do NOT implement tracks yet."
- `EditorAiCopilot`'s Recent AI Tasks section is empty by default (no backend) — the list rendering and status iconography are ready for a future sprint to populate.
- Resizing/collapsing works via mouse/pointer and arrow keys on the divider itself; touch-drag was not specifically tested (desktop-only scope, per spec).
- No virtualization was added to the asset grid yet (only 6 mock assets) — the brief asked only to "prepare for" virtualization, not implement it.
