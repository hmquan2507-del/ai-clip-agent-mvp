# Sprint 16.10.6 — AI Decision Action Runtime

## Status

Implemented as a frontend action layer with an optional runtime adapter. No backend action contract was invented.

## Architecture

```text
DesktopEditorShell
  └─ AiDecisionActionProvider
       ├─ AiTimeline
       │    ├─ selectDecision(decision)
       │    └─ runAction(action, decision)
       └─ EditorInspector
            └─ AiDecisionInspector
```

The action provider owns decision lifecycle presentation state, action status, selected decision, inline errors, and decision history. It does not import Timeline Runtime, Playback Runtime, History Runtime, Selection Runtime, backend clients, workers, or database code.

## Decision lifecycle

Supported states:

- draft
- applied
- accepted
- rejected
- disabled
- pinned
- manual
- regenerating
- processing
- stale
- error

## Actions

The inspector exposes:

- Accept
- Reject
- Regenerate
- Disable / Enable contract
- Pin / Unpin contract
- Duplicate contract
- Convert to Manual
- Explain contract
- Compare contract

Actions are emitted through a strongly typed `AiDecisionActionAdapter`. Without an adapter, the UI explicitly reports `Pending Runtime`; it never pretends that the backend or timeline changed.

## Known limitations

- No backend decision-action endpoint exists in the supplied frontend source.
- Action states are session-local and reset on reload.
- Compare and Explain use the existing decision metadata only.
- Bulk actions are deferred until AI decision multi-selection has a stable shared contract.
- Successful action confirmation and authoritative timeline refresh require a future backend adapter.
