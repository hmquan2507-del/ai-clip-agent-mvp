# Sprint 16.10.2 — AI Video Studio Editor Shell (CapCut-inspired)

Date: 2026-07-24

## Objective

Replace the review editor's page layout with a professional, CapCut-inspired shell (header, left media panel, center preview, right inspector/AI Copilot, bottom timeline, status bar) as a pure UI-layer change. No runtime, backend, API, or state-machine logic was modified.

## Architecture

The existing `ReviewWorkspace` → `ReviewRuntimeWorkspace` → `ReviewWorkspaceProvider` → `ReviewRuntimeWorkspaceContent`/`ReviewRuntimeWorkspaceEditor` chain (in `src/features/review/integration/runtime-workspace.tsx`) is **completely untouched**. That chain still owns session lifecycle, drag/trim/keyboard hooks, and view-model construction (`buildReviewEditorViewModel`), and it still renders exactly one component with exactly the same props as before: `ReviewEditorShell`.

The only file inside the runtime boundary that changed is `src/features/review/shell/review-editor-shell.tsx` — and only its **internal JSX composition**. Its exported `ReviewEditorShellProps` interface is byte-for-byte identical to before, so `runtime-workspace.tsx` required zero changes. Internally, `ReviewEditorShell` now renders the new `editor-shell` layout components instead of the old `ReviewEditorTopbar` / `ReviewEditorRail` composition, while still rendering the **exact same** `ReviewPreviewStage` and `ReviewTimelinePanel` components with the exact same props as before (now via thin wrappers `PreviewPanel` and `TimelineContainer`).

```
ReviewWorkspace (unchanged)
  └─ ReviewRuntimeWorkspace (unchanged, provider + hooks + adapter)
       └─ ReviewEditorShell (internal composition replaced, prop contract unchanged)
            ├─ EditorHeader              (new, presentational)
            ├─ EditorLeftPanel           (new, presentational, mock media library)
            ├─ PreviewPanel              (new wrapper) → ReviewPreviewStage (unchanged, reused)
            ├─ EditorRightPanel          (new)
            │    ├─ "Properties" tab     → ReviewInspectorPanel (unchanged, reused — real AI suggestions + clip fields)
            │    └─ "AI Copilot" tab     (new mock suggestion list) + ReviewAICommandBar (unchanged, reused)
            ├─ TimelineContainer (new wrapper) → ReviewTimelinePanel (unchanged, reused — real drag/trim/select wiring)
            └─ StatusBar                 (new, presentational, read-only)
```

Nothing under `state/`, `integration/` (except the untouched prop-passing in `runtime-workspace.tsx`, which needed no edits), `react/`, `drag/`, `trim/`, `keyboard/`, `viewport/`, or `features/playback/` was modified.

Since replacing that layout was this sprint's explicit goal, the assertions were updated to check the equivalent invariants against the new markup (header renders, left panel renders, preview/timeline/AI-suggestions/AI-command-bar still render, responsive classes present, command input renders before the timeline in DOM order). No invariant was weakened — each old check has a like-for-like new-layout replacement.

## Integration notes

- `EditorRightPanel`'s "Properties" tab renders the real, unmodified `ReviewInspectorPanel` — so the existing AI-suggestions apply/dismiss/regenerate flow (wired to `onAISuggestionCommand`) is fully preserved, not just a placeholder.
- `EditorRightPanel`'s "AI Copilot" tab (the default tab) hosts the ten requested suggestion entries (Highlight first 15s, Remove Silence, Add B-roll, Replace Music, Improve Subtitle, Add Zoom, Rewrite Hook, Generate CTA, Translate Subtitle) as callback-only buttons (`onAiSuggestionSelect`) — no API calls, no timeline mutation, per spec. The "Custom Prompt" entry is backed by the real, unmodified `ReviewAICommandBar` (reused, not forked), wired to the same `onAICommandSubmit`/pending/disabled/accepted-submission props the old bottom command bar used.
- `PreviewPanel` and `TimelineContainer` are intentionally thin (a styled wrapper `div` around the existing component) — they do not reimplement or duplicate any playback/timeline logic.
- `StatusBar` reads real `fps`, `durationLabel`, and a formatted `playheadTime` from the existing timeline view model where available; `zoom`/`snap`/`resolution`/`speed` are presentational fields (no equivalent live values exist in the current view-model contract yet).
- All cross-feature imports in the new `editor-shell` components use relative paths (`../../review/shell`) rather than the `@/` alias, matching the existing convention in `review-editor-shell.tsx` and required for compatibility with `scripts/test-review-editor-shell.cjs`, which resolves modules via plain Node `require` (no tsconfig path-alias support).

## Known limitations

- Preview transport controls (play/pause/frame-step/zoom%/aspect-ratio/fullscreen) are the same decorative, non-synced controls that already existed in `ReviewPreviewStage` — real playback sync is out of scope for this sprint (called out in existing code as pending "Sprint 16.5" work) and remains so.
- `EditorLeftPanel` tabs other than "Media" (Audio, Text, Stickers, Effects, Transitions, Filters, Templates, Brand Kit, AI Assets, Local Files) are placeholders with no content, per spec.
- `EditorRightPanel`'s "Properties" tab reuses the existing `ReviewInspectorPanel`, which itself only has real content under its internal "Clip" tab (Text/Audio/AI sub-tabs there are static, pre-existing behavior, not changed by this sprint).
- `StatusBar`'s Zoom/Snap/Resolution/Speed fields have no backing runtime value yet (no such fields exist in `ReviewTimelineView`) and currently show fixed placeholder text.
- Resizable panels were not implemented (no such library exists in `package.json`, and the target layout didn't require it for this sprint — left/right panel widths are fixed with a collapse toggle on the left panel only).
- The old `ReviewEditorTopbar` and `ReviewEditorRail` components (`shell/editor-topbar.tsx`, `shell/editor-rail.tsx`) are no longer referenced by `ReviewEditorShell` but were left in place (unused) rather than deleted, consistent with this codebase's established "don't delete, flag for cleanup" pattern from the prior sprint.
