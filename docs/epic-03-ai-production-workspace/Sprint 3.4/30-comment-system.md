## Purpose

Define comments inside Review Workspace.

Comments help users capture review notes without turning the product into a timeline editor.

## Comment Types

Supported comments:

- timestamp comment
- clip comment
- transcript comment
- subtitle comment
- general production comment

## Comment Fields

Each comment includes:

- author
- timestamp
- target
- text
- status
- created time
- resolved time

## User Actions

Users can:

- add comment
- edit comment
- resolve comment
- reopen comment
- jump to target

## Status

Comment status:

- Open
- Resolved
- Reopened

## Non-Goals

Comments are not a full collaboration system in the MVP.

Team review workflows can be added in SaaS phases.
