# Sprint 16.8.4 — Export Workspace Router Integration & Render Status API

## Status

Implemented.

## Goal

Mount the Export Workspace API in the root FastAPI application and expose a read-only render status endpoint backed by the existing queue runtime.

## Endpoints

- `POST /api/v1/export-workspace/render-submissions`
- `GET /api/v1/export-workspace/render-submissions/{queue_job_id}`
- `GET /api/v1/export-workspace/health`

## Status response

The render status projection exposes:

- queue job id
- production id
- queue type
- status
- progress
- error
- created, started, finished, and updated timestamps

## Architecture

```text
FastAPI root app
        ↓
Export Workspace router
        ├── submission application service
        │       ↓
        │   RenderJobSubmissionRuntime
        │
        └── status projection service
                ↓
            QueueService.get
```

The status endpoint is read-only and does not mutate the queue job.

## Integration strategy

The installer executes a Python integration script that performs an idempotent, anchor-validated update of `app/main.py`. PowerShell does not patch Python source code.
