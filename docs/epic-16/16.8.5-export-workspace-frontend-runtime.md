# Sprint 16.8.5 — Export Workspace Frontend Runtime

## Status

Implemented.

## Goal

Connect the frontend Export Workspace to the backend render submission and render status APIs without coupling UI components directly to `fetch`.

## Runtime flow

```text
ReviewRenderContract
        ↓
ExportRuntimePanel
        ↓
useExportWorkspaceRuntime
        ↓
ExportWorkspaceRuntime
        ↓
ExportWorkspaceApiClient
        ↓
POST render-submissions
        ↓
GET render-submissions/{queue_job_id}
```

## Added frontend layers

- typed API contracts
- API client with injectable `fetch`
- observable runtime state machine
- render submission
- render status polling
- terminal status handling
- cancellation and unmount cleanup
- React `useSyncExternalStore` hook
- reusable Export Runtime panel

## Runtime phases

- idle
- submitting
- queued
- polling
- completed
- failed
- cancelled

## Environment

The API client uses:

```text
NEXT_PUBLIC_API_URL
```

and defaults to:

```text
http://localhost:8000
```

## Integration boundary

Existing Export pages can import:

```ts
import { ExportRuntimePanel } from "@/features/export-workspace";
```

and pass the immutable `ExportRenderContract` generated by the Review Workspace.
