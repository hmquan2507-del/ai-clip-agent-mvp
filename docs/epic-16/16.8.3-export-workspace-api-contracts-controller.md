# Sprint 16.8.3 — Export Workspace API Contracts & Controller

## Status

Implemented.

## Goal

Expose the immutable Review-to-Render handoff and Sprint 16.8.2 submission runtime through a stable HTTP application boundary for the Export Workspace frontend.

## API

### POST `/api/v1/export-workspace/render-submissions`

Accepts a complete immutable `ReviewRenderContract` plus optional request metadata.

Returns HTTP `202 Accepted` with:

- queue job id
- production id
- snapshot id
- timeline revision
- idempotency key
- duplicate flag

Domain/runtime failures remain explicit in the response body through `success`, `error_code`, and `error`.

### GET `/api/v1/export-workspace/health`

Lightweight component health endpoint.

## Architecture

```text
Frontend Export Workspace
        ↓
FastAPI Controller
        ↓
ExportWorkspaceApplicationService
        ↓
RenderJobSubmissionRuntime
        ↓
QueueService adapter
        ↓
RENDER_RUNTIME queue
```

The API never reads `TimelineRuntimeStore`. It accepts only the immutable contract created by Sprint 16.8.1.

## Integration

Use `create_export_workspace_api_router(queue_service_provider)` and include the returned router in the root FastAPI application composition layer. This sprint intentionally avoids editing the project root application file because repositories may compose routers differently.

## Tests

`backend/scripts/test_export_workspace_api_contracts_controller.py`
