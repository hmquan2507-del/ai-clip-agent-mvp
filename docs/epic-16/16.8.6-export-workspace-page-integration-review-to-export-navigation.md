# Sprint 16.8.6 — Export Workspace Page Integration & Review-to-Export Navigation

## Status

Implemented.

## Goal

Connect the Review Workspace export action to the real Export Workspace page while preserving the immutable render contract created by the Review-to-Render handoff runtime.

## Navigation flow

```text
Review Workspace authoritative snapshot
        ↓
extract immutable render contract
        ↓
sessionStorage versioned envelope
        ↓
/export?production_id=...&source=review
        ↓
Export Workspace page
        ↓
ExportRuntimePanel
        ↓
render submission + status polling
```

## Safety rules

- The page never rebuilds or mutates the render contract.
- Contracts are accepted only after structural validation.
- A production ID mismatch rejects stale stored contracts.
- Missing contracts produce a safe empty state.
- The Review export action is disabled until a valid contract exists.
- Back navigation preserves the production ID.

## Storage boundary

The browser transfer uses a versioned `sessionStorage` envelope. This keeps the immutable payload out of the URL while allowing a route transition in the same browser tab.
